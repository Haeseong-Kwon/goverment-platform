# 1\. 제품 개요

## 1.1 한 줄 정의

지원사업(예창패·초창패·모두의창업 등)을 준비하는 창업자에게 준비 → 진단 → 팀 협업 → 선정 전환까지의 전 과정을 하나의 워크스페이스로 제공하는 서비스.

## 1.2 설계 원칙 3가지

- A1 리포트 문법 통일 - 모든 AI 진단은 "점수 → 기여 분해 → 판단 근거 → 미확인 데이터 고지 → 다음 할 일" 구조로 출력한다. 신뢰(정직 고지)와 실행(액션 연결)을 동시에 담보한다.
- 가시성 분리(RBAC) - 창업자의 준비 데이터(TODO·진단 점수·초안·보관함)는 매니저에게 절대 비노출. 창업자에게는 연습장, 매니저에게는 결과만.
- 자산 축적형 락인 - 쓸수록 프로필·서류·진단 이력이 쌓여 재지원 시 자동 재사용되는 구조. 이탈 비용이 시간에 비례해 증가한다.

## 1.3 역할 전환 구조

pre_founder (선정 전, 무료)

│

└── \[P8\] 기관 코드 입력 ──▶ founder (선정 후, 기관 번들)

└ 정산 사전검증 · 상태 트래커로 확장

# 2\. 화면별 기능 및 효과

## \[P0\] 진입 분기

| **기능**          | 창업자/매니저 역할 선택 2분할 진입 화면                            |
| ----------------- | ------------------------------------------------------------------ |
| **사용자 효과**   | 역할별로 완전히 다른 제품 경험을 첫 화면에서 분리 - 혼선 제거      |
| **비즈니스 효과** | B2C 유입(창업자)과 B2G 세일즈(매니저 기관 인증)를 한 랜딩에서 처리 |

## \[P1\] 온보딩 (프로필 → 지원사업 → 아이템)

| **기능**          | 이름·포지션 입력, 준비 중인 지원사업 멀티 선택, 아이템 소개, 팀원 초대 링크. 팀빌딩 희망 여부·찾는 포지션 필드 포함(등록만)              |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | 3분 안에 개인화 세팅 완료 - 이후 모든 화면(D-day·TODO·진단·추천)의 데이터 원천                                                           |
| **비즈니스 효과** | ① 지원사업 선택 = 자동 마일스톤·캘린더·자격진단 트리거 ② 팀빌딩 필드 = 향후 매칭 기능의 공급 풀 사전 축적 ③ 초대 링크 = 바이럴 계수 확보 |
| **핵심 설계**     | 온보딩 완료 즉시 자격 사전진단(P4-1) 자동 실행 - 첫 세션에서 "아하 모먼트" 전달                                                          |

## \[P2\] 홈 대시보드

| **기능**          | D-day 히어로 카드(가장 임박한 공고), 오늘의 TODO, 자격진단 요약, 팀 진행률, 빠른 계산기, 창업 상식 아티클 |
| ----------------- | --------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | "오늘 뭘 해야 하는가"를 로그인 10초 안에 파악                                                             |
| **비즈니스 효과** | 재방문의 착지점. D-day와 미완료 TODO가 매일의 복귀 이유를 생성                                            |

## \[P3\] 팀 TODO 보드

| **기능**          | 칸반 3열(할 일/진행/완료). 지원사업 선택 시 마감 역산 마일스톤 자동 생성(사업계획서 초안 D-14, 증빙 준비 D-10, 리허설 D-7, 제출 D-1). 담당자 지정, D-day pill, 작업 단위 코멘트 스레드 |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | "뭘 언제까지 해야 하는지" 몰라서 생기는 준비 실패를 구조적으로 방지. 팀원 간 진행 공유로 확인 요청 감소                                                                                |
| **비즈니스 효과** | ① 자동 마일스톤 = 노션·일반 TODO 앱과의 차별점(도메인 지식 내장) ② 팀원 계정 묶임 = 팀 단위 락인 ③ 코멘트 스레드가 채팅 수요를 흡수하되 개발비는 1/10                                  |
| **의도적 제외**   | 실시간 팀 채팅 - 카톡과 경쟁 불가 판단, 업무 객체 종속 코멘트로 대체                                                                                                                   |

## \[P7\] 마감 캘린더

| **기능**          | 월간 캘린더 + 다가오는 일정 리스트. 이벤트 3종(공고 마감 red / 자동 마일스톤 blue / 커스텀 slate), 알림 |
| ----------------- | ------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | 복수 지원사업의 마감 충돌을 한눈에 - "마감을 몰라서 놓치는" 사고 제거                                   |
| **비즈니스 효과** | 시즌 리듬 기반 재방문 트리거(알림 = 리텐션 채널). 추천 사업의 \[캘린더에 추가\]가 유입되는 종착지       |

## \[P4-1\] 자격 사전진단 리포트

| **기능**          | 창업 이력·기수혜·대표자 요건 등 기반 "자격 적합도 점수 + 결격 위험 사전 적발". 하단에 지원 가능 사업 AI 추천(RecommendationCards) - 적합 근거·공고 시기·캘린더 연결 |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | 서류 실격이라는 정보 부족형 실패를 신청 전에 차단. 내 조건으로 갈 수 있는 다른 사업까지 발견                                                                        |
| **비즈니스 효과** | ① 경쟁 공백 영역(실격 예방 전문 서비스 부재) ② 온보딩 직후 첫 가치 전달 ③ 추천 → 캘린더 → TODO로 이어지는 제품 내 순환                                              |
| **리스크 방어**   | "아직 확인하지 못한 데이터" 섹션 + "AI 추정, 공고문 기준" 고지 필수 - 오탐 신뢰 리스크 방어막                                                                       |

## \[P4-2\] 사업계획서 AI 진단 + SWOT

| **기능**          | 사업계획서 업로드 → 합격 준비도 점수 + PSST 4축 분해(문제인식/실현가능성/성장전략/팀) + 약한 섹션 지적 + SWOT 4분면(마케팅 에이전트 A1 모듈 재사용) + 진단 이력 추이(v1 48 → v2 62) |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | 유료 컨설팅 없이 보완 방향 확보. 버전별 점수 추이로 개선을 체감                                                                                                                     |
| **비즈니스 효과** | ① 무료 월 2회 + 초대 시 추가 = 그로스 루프 ② 진단 이력 = 자산 축적형 락인의 핵심 데이터 ③ A1 모듈 재사용으로 개발비 절감                                                            |
| **포지셔닝 경계** | "작성 대행"이 아닌 "진단" - 대필 브로커 규제·평판 리스크 회피                                                                                                                       |

## \[P5\] 세금 계산기 (3종)

| **기능**          | 4대보험 실부담액 / 법인 vs 개인 세금 비교 / 인건비 총부담액. 입력 3~4개 → 항목 분해 결과. 비로그인 사용 가능, PDF 저장 시 이메일 수집 |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | "직원 뽑으면 실제 얼마 나가나"를 30초 만에 - 나홀로 창업자까지 커버하는 유일한 범용 유틸리티                                          |
| **비즈니스 효과** | 검색 유입형 리드마그넷(SEO 착지 페이지). 이메일 수집 = 합격 시즌 온보딩 타깃 리스트                                                   |
| **법적 경계**     | "참고용, 신고는 세무 전문가 확인" 고지 - 세무사법 경계 준수. 개별 상담형 답변 미제공                                                  |

## \[P6\] 서류 보관함 (내 서류 / 무료 자료실)

| **기능**          | 내 서류: 고정 폴더(사업계획서/증빙/제출 아카이브) + 버전 태그. 자료실: 감수·출처 표기된 표준 계약서(동업계약서, KVCA 표준투자계약서, NDA, IR 템플릿 등) 다운로드 |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | 지원 서류가 흩어지지 않고 버전 관리됨. 비싼 계약서 양식을 신뢰 가능한 출처로 무료 확보                                                                           |
| **비즈니스 효과** | ① 선정 시 서류가 정산 시스템으로 그대로 이관 - "여기 쌓아두면 이득"인 구조 = 락인 ② 자료실 다운로드 = 이메일 수집 ③ 자료실 → 전문가 상담(P9) 연결 리드           |
| **의도적 제외**   | 범용 파일 드라이브(임의 폴더·공유 링크) - 협업툴 경쟁 회피, "제출 서류 vault"로 한정                                                                             |

## \[P9\] 법인 설립 가이드

| **기능**          | 설립 절차 5단계 스텝퍼(정관 → 자본금 → 등기 → 사업자등록 → 4대보험) + 단계별 서류 체크리스트 + 프로필 연동 시점 경고(예창패 준비자에게 "신청 전 설립 시 자격 상실 위험" 경고) + 제휴 법무법인 상담 신청 폼 |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | 설립 절차 정보 격차 해소 + "언제 설립해야 하는가"라는 자격 연동 판단까지 제공 - 단순 정보 페이지와의 결정적 차이                                                                                           |
| **비즈니스 효과** | 나홀로 창업자 트랙 커버 + 법무 상담 리드 생성                                                                                                                                                              |
| **법적 경계**     | 변호사 소개 대가 수수 금지(변호사법 제34조) - 마켓플레이스·수수료 구조 배제, 광고 표기된 상담 신청 폼만 운영                                                                                               |

## \[P11\] 커넥트 허브 (팀빌딩 / 멘토 / 투자 - 프리뷰)

| **기능**          | 3탭 Coming Soon 프리뷰(블러 카드 + 대기 신청). 팀빌딩은 기관 프로그램 연동형으로 재정의 - 기관이 개설한 프로그램(예: 인하대 "나랑 팀 해 듀오") 카드에 신청, 매칭 파이프라인은 기관 주도 |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | 로드맵 가시화 - "여기서 팀·멘토·투자까지 이어진다"는 기대 형성. 기관 프로그램 참여 창구 일원화                                                                                          |
| **비즈니스 효과** | ① 대기 신청 수 = 개발 착수 여부의 정량 기준(수요 검증 실험) ② 기관 주도 구조로 콜드스타트 해소 ③ 4대보험 계산기를 신청 폼에 임베드하는 크로스셀                                         |

## \[P8\] 합격 전환 플로우

| **기능**          | \[합격하셨나요?\] → 기관 코드 입력 → 팀·서류·프로필 그대로 founder 워크스페이스로 승격(정산 사전검증·상태 트래커 활성화)           |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| **사용자 효과**   | 합격 후 "이제 뭘 하지"의 공백 없이 협약 준비로 연속 이행. 쌓아둔 자산 재입력 불필요                                                |
| **비즈니스 효과** | 선정 전 무료 제품과 B2G 유료 제품을 잇는 다리 - 전체 비즈니스 모델의 존재 이유. 합격자 명단이 자동으로 기관 온보딩 파이프라인이 됨 |

# 3\. 전체 워크플로우

가입(P0) → 온보딩(P1) → 자격진단 자동실행(P4-1)

→ \[매일\] 홈(P2) ← D-day · TODO(P3) · 캘린더(P7) 리듬

→ \[준비\] 사업계획서 진단 + SWOT(P4-2) ↔ 보관함(P6) 버전 축적

→ \[상시\] 계산기(P5) · 법인 가이드(P9) · 자료실(P6) · 커넥트(P11)

→ \[합격\] 전환(P8) → founder: 정산 사전검증 · 상태 트래커 → 매니저 파이프라인 연결

→ \[불합격\] 진단 이력 · 서류 자산 보존 → 추천 사업(P4-1) 재지원 루프

# 4\. KPI 맵

기능마다 측정 지표와 성격을 매핑한다. 개발 시 이벤트 로깅 설계의 기준이 된다.

| **기능**              | **핵심 지표**                                        | **성격**      |
| --------------------- | ---------------------------------------------------- | ------------- |
| P1 온보딩             | 완료율, 팀원 초대율                                  | 활성화        |
| P4-1 자격진단         | 온보딩→진단 완료율, 추천 카드 \[캘린더 추가\] 클릭률 | 아하 모먼트   |
| P3 / P7               | 주간 재방문율(WAU), TODO 완료율                      | 리텐션        |
| P5 계산기 / P6 자료실 | 비로그인 유입 수, 이메일 전환율                      | 리드          |
| P4-2 사업계획서 진단  | 월 진단 횟수, 초대 경유 크레딧 사용률                | 그로스 루프   |
| P11 커넥트            | 탭별 대기 신청 수 (팀빌딩 기준선: 4주 50명)          | 수요 검증     |
| P8 합격 전환          | 합격 전환율, 전환 후 기관 연결율                     | 비즈니스 전환 |

# 5\. 의도적으로 만들지 않은 것 (판단 기록)

| **제외 항목**                  | **사유**                                                     |
