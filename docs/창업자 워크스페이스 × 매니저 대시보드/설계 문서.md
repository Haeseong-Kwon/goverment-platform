## 1. 제품 정의 & 역할 구조

**한 줄 정의:** 지원사업(예창패·초창패·모두의창업 등)을 준비하는 창업자에게는 "준비 워크스페이스"를, 선정 팀을 관리하는 주관기관 매니저에게는 "통합 관리 대시보드"를 제공하는 이원화 SaaS.

### 역할 3종 (RBAC)

| 역할 | 정의 | 진입 |
|---|---|---|
| `pre_founder` | 지원사업 준비 중 (선정 전) | 랜딩 → [창업자로 시작] → 온보딩 |
| `founder` | 선정 후 협약 수행 중 | pre_founder가 합격 전환 or 기관 초대 링크 |
| `manager` | 주관기관 담당자 | 랜딩 → [매니저로 시작] → 기관 인증 |

### 가시성 절대 규칙 (모든 화면 적용)

1. 창업자의 준비 데이터(TODO, 진단 점수, 초안, 보관함)는 **매니저에게 절대 노출되지 않는다**
2. 매니저는 `founder`의 "검증 통과 후 검토 요청 건"만 본다 ([E][F] 플로우)
3. 팀원(pre_founder 동일 팀)끼리는 TODO·진행 상황·보관함 공유 가능
4. 진단 리포트의 점수는 본인 팀 내부에서만 공유

## 3. 공통 컴포넌트

### 3.1 A1 리포트 문법 (진단 계열 공통 — 순서 고정)

모든 AI 진단 리포트는 다음 순서를 강제한다:

① `<ScoreRing>` — 점수 링(conic-gradient) + 상태 뱃지 + "AI 추정" 뱃지
② `<ContributionBars>` — 기여/항목 분해 막대 (**막대는 blue 계열 전용**, 미확정 항목은 dashed 트랙 + 상태 뱃지)
③ `<ReasonList>` — "왜 이렇게 판단했나요" 근거 리스트 (✓ green / ! amber·red 원형 아이콘)
④ "아직 확인하지 못한 데이터" — 정직 고지 섹션 (○ 아이콘, slate-50 카드)
⑤ `<NextActions>` — 체크리스트 + [TODO에 추가] Secondary 소형 버튼
⑥ 고지 Caption — "본 진단은 참고용 AI 추정이며…" (13/500, slate-400)

추가 모듈:
- `<RecommendationCards>` — 지원 가능 사업 AI 추천 카드 3장 (적합/확인 필요 뱃지 + 공고 시기 + [캘린더에 추가])
- `<SwotQuadrant>` — SWOT 4분면 (S green / W amber / O blue / T red 뱃지) — **마케팅 에이전트 A1 모듈과 동일**

재사용 매트릭스:

| 컴포넌트 | P4-1 자격 | P4-2 사업계획서 | P5 계산기 | SWOT |
|---|---|---|---|---|
| ScoreRing | ✓ | ✓ | — | — |
| ContributionBars | ✓ | ✓ (PSST) | ✓ (보험료 분해) | — |
| ReasonList | ✓ | ✓ | — | — |
| NextActions | ✓ | ✓ | — | — |
| RecommendationCards | ✓ | — | — | — |
| SwotQuadrant | — | ✓ | — | ✓ |

### 3.2 기타 공통 컴포넌트

- **D-day pill:** D-8 이상 slate / D-7~4 amber / D-3 이하 red — 전 화면 동일 로직
- **상태 뱃지 4종:** 완료(green) / 진행·임박(amber) / 반려·위험(red) / 초안·중립(slate)
- **TODO 카드:** 체크박스 + 담당자 아바타 + 마감 pill + 코멘트 카운트 + `자동 생성` 뱃지(blue-50)
- **코멘트 스레드:** 객체(TODO·파일)에 종속 — 채팅 UI 아님, 알림은 멘션 기반
- **파일 카드:** 파일명 + 버전 태그(v1·v2·최종) + 업로더 아바타 + 날짜 + 코멘트 카운트
- **이메일 수집 모달:** P5 계산기 PDF 저장·P6 자료실 다운로드에서 동일 모달 재사용 (리드 KPI)
- 캘린더(월간)·스텝퍼·토스트(#0F172A 배경)·empty state·업로드 드롭존·데이터 테이블

## 6. 인터랙션 명세 (개발 핸드오프)

1. **D-day pill 로직:** D-8 이상 slate / D-7~4 amber / D-3 이하 red — 전 화면 공통 함수
2. **자동 마일스톤:** 지원사업 선택 시 서버가 템플릿 생성(D-14 초안 완성 / D-10 증빙 준비 / D-7 리허설 / D-1 최종 제출). TODO 카드에 `자동 생성` 뱃지, **삭제 대신 "숨기기"**
3. **진단 무료 횟수:** 월 2회, 매월 1일 리셋. 소진 시 P4-2 업로드 비활성 + "팀원 초대 시 1회 추가"
4. **P5 [PDF로 저장] = 이메일 수집 이벤트** (전환 KPI 포인트). P6 자료실 [다운로드]도 동일 모달 재사용
5. **P8 전환:** 기관 코드 검증 → role `pre_founder → founder`, PrepTeam → Team 이관, 사이드바 즉시 교체. 이관 대상은 **서류 보관함·팀원만** — TODO·진단 점수·초안은 팀 내부 잔류
6. **P9 배너 조건부:** 예창패 준비 프로필 → red 경고("신청 시 사업자등록 없어야…") / 나홀로 창업자 → blue-50 안내
7. **P11 커넥트:** 3탭 전부 Coming Soon 프리뷰 — 탐색/검색/프로필 상세/메시지 UI 없음. 등록은 P1 스텝1 팀빌딩 필드 재사용
8. **매니저 [F]:** 반려는 사유 코드 필수. AI 사전 검증은 "검증 통과 🟢" 뱃지만 — **점수 숫자 미표기**

## 8. 품질 가드레일

- 텍스트 대비 4.5:1 이상 · Pretendard · 아이콘 라인 스타일 통일
- 창업자 = 여백·친절 / 매니저 = 밀도·스캔 — 같은 제품임이 느껴지게
- 상태 컬러는 뱃지·pill·게이지 전용 (버튼·차트 금지, 단 Destructive 반려 버튼 예외)
- 모든 진단·계산 화면에 "AI 추정/참고용" 고지 Caption 필수 (세무 고지는 red + ⚠️)
- **"제출" 워딩 금지** → "검증 / 검토 요청 / 제출 준비 완료"
- 실제 한글 카피·실데이터 필수, 로렘입숨 금지
- 다크모드·과한 그라데이션 금지, 블루+화이트 주도
