## 0. 제품 한 줄 요약

**K-Startup 예비창업패키지(예창패)** 정부지원사업의 **서류 준비·정산 제출·검증 프로세스를 제출 전에 "예행연습"** 시켜, 반려를 사전에 차단하는 **AI 행정 시뮬레이터**.

- **핵심 가치**: "예창패 서류, 제출 전에 예행연습하세요."
- **문제**: 카톡·이메일로 흩어진 서류 검토 → 잦은 반려 → 시간 낭비.
- **해결**: AI가 필요 서류 자동 도출 + 비목 사전검증 + 회계사·매니저 **이중 검증 게이트**.

### 3개 사용자 역할(Role)
| 역할 | 색상 코드 | 담당 화면 |
|---|---|---|
| **창업자(Founder)** | 파랑 `#2563EB` | B 온보딩, C 정산 제출, D 상태 트래커 |
| **매니저(Manager)** | 보라 `#7C3AED` | E 통합 대시보드, F 검토 큐 |
| **회계사(Accountant)** | 남색 `#0F172A` (뱃지는 파랑) | G 제한 검증 뷰 |

### 전체 화면(아트보드) 목록 — 7개
| ID | 이름 | 역할 | 상태/인터랙션 |
|---|---|---|---|
| **A** | 랜딩 (공개) | 공개 | 정적 |
| **B** | AI 시뮬레이터 온보딩 (3스텝) | 창업자 | 스텝 상태머신 |
| **C** | 정산 제출 폼 | 창업자 | 비목 선택 → 증빙 목록 전환 |
| **D** | 정산 상태 트래커 | 창업자 | 정적(파이프라인 표시) |
| **E** | 통합 대시보드 | 매니저 | 정적(도넛·표) |

| **G** | 제한 검증 뷰 | 회계사 | 정적(미니멀) |

## 2. 상태 관리 (State Machine)

원본은 하나의 컴포넌트에 3개의 독립 상태를 둔다. 페이지 분리 시 각 페이지의 로컬 상태로 옮기면 된다.

```
state = { step: 1, account: 'promo', q: 'q1' }
```

| 상태 | 값 | 사용 화면 |
|---|---|---|
| `step` | `1 | 2 | 3` | **B 온보딩** — 현재 스텝 |
| `account` | `'promo' | 'machine' | 'labor'` | **C 정산 제출** — 선택된 비목(→ 증빙 목록 전환) |
| `q` | `'q1' | 'q2' | 'q3' | 'q4'` | **F 검토 큐** — 선택된 큐 항목(→ 우측 상세) |

### 2.1 B 온보딩 스텝 로직
- `nextStep`: `step = min(3, step+1)`
- `prevStep`: `step = max(1, step-1)`
- `toStep1/2/3`: 스텝 인디케이터 클릭 시 해당 스텝으로 직접 이동.
- **스텝 도트 스타일** `stepDot(n)`:
  - `done = n < step`, `active = n === step`
  - 크기 `38×38`, 원형, `font-weight 800 / 15px`
  - done·active → 배경 `#2563EB`, 글씨 흰색, 테두리 없음
  - 그 외 → 배경 흰색, 글씨 `#94A3B8`, 테두리 `2px solid #E2E8F0`
- **스텝 라벨 스타일** `stepLbl(n)`: active → `700 / #0F172A`, 아닐 때 → `500 / #94A3B8`.
- 각 스텝 카드는 `step` 값에 따라 하나만 표시(조건부 렌더).

### 2.2 C 정산 제출 비목 로직
- 비목 칩 `chip(key)`: 선택됨(`account===key`) → 배경 `#2563EB`/흰 글씨/테두리 `1px solid #2563EB`; 아니면 흰 배경/`#475569`/`1px solid #E2E8F0`. `transition:all .15s`.
- `selPromo/selMachine/selLabor` → `account` 변경.
- 우측 **증빙 체크리스트**는 `account`에 따라 3개 목록 중 하나만 표시(§4.3 데이터).

### 2.3 F 검토 큐 로직
- 큐 행 `qRow(id)`: 선택됨 → 좌측 `3px solid #2563EB` 보더 + 배경 `#EFF6FF`; 아니면 투명 보더 + 흰 배경. `transition:all .15s`.
- `selQ1~selQ4` → `q` 변경.
- 우측 상세 패널의 제목/금액/비목은 `q` 값으로 매핑(§5.2 데이터).

## 3. 화면 A — 랜딩 (공개)

**컨테이너**: 폭 `1440px`, 흰 배경, radius `20px`, overflow hidden. 섹션 순서대로:

### 3.1 헤더 (sticky)
- `sticky; top:0; background:rgba(255,255,255,.9); backdrop-filter:blur(8px); z-index:10;` 하단 `1px solid #E2E8F0`. 패딩 `20px 56px`.
- **로고**: `34×34` 파랑(`#2563EB`) radius `9px` 사각형 + 흰 체크 아이콘, 옆에 `행정 시뮬레이터` (`17px/800`, letter-spacing `-.02em`).
- **내비**(`15px/600`, `#475569`): 서비스 · 지원 프로그램 · 요금 · 고객센터.
- **우측**: `로그인`(텍스트) + `무료로 시작`(파랑 버튼 `11px 20px`, radius `10px`).

### 3.2 히어로
- 2컬럼 그리드 `1.05fr .95fr`, gap `56px`. 배경 `linear-gradient(180deg,#F8FAFC,#fff)`. 패딩 `76px 56px 68px`.
- **좌측**:
  - 알약 뱃지: `K-Startup 예비창업패키지 대응` (배경 `#EFF6FF`, 글씨 `#2563EB`, `13px/700`).
  - H1: `예창패 서류,<br>제출 전에 [예행연습]하세요` — "예행연습"만 `#2563EB`. (`44px/700`)
  - 본문: `AI가 필요한 서류를 자동으로 찾아주고, 반려로 이어질 문제를 제출 전에 미리 잡아드립니다.` (`18px/#475569`, max-width `480px`)
  - CTA 2개: `무료로 시뮬레이션 시작`(파랑, 그림자 `0 8px 22px rgba(37,99,235,.28)`) + `매니저이신가요? →`(흰 배경, 파랑 글씨/테두리 `1.5px solid #2563EB`).
- **우측 — 필요 서류 체크리스트 카드**(흰 카드, 테두리+그림자, padding `22px`):
  - 헤더: `필요 서류 체크리스트`(`14px/700`) + `자동 생성` 알약(파랑 tint).
  - 체크 항목 3개(각 상단 `1px solid #F1F5F9` 구분선, `12px 0`):
    1. `사업자등록증 사본` — `20px` 초록 원(`#16A34A`) + 흰 체크, 우측 `완료`(초록).
    2. `견적서 2부 (기계구입비)` — `20px` 주황 원(`#F59E0B`), 우측 `준비중`(주황).
    3. `4대보험 가입자명부` — `20px` 회색 원(`#E2E8F0`), 텍스트 `#475569`, 우측 `대기`(`#94A3B8`).
  - 하단 경고 박스: 배경 `#FEF2F2`, 테두리 `#FECACA`, 경고 아이콘 + `반려 위험 2건 — 제출 전 확인하세요` (`12.5px/#B91C1C`).
  - **떠 있는 배지**(카드 좌하단 `bottom:-18px;left:-18px`): 다크(`#0F172A`) 박스, `예상 반려 사유`(회색 캡션) + `증빙 누락 · 한도 초과`(`15px/700` 흰색).

### 3.3 통계 밴드
- 3컬럼(`repeat(3,1fr)`), 상하 `1px` 테두리, 배경 `#F8FAFC`, 컬럼 사이 우측 테두리. 각 셀 패딩 `38px 40px`, 가운데 정렬.
- 숫자 `40px/800/#2563EB`, 캡션 `15px/#475569/600`:
  - `6종` — 사전 차단하는 반려 유형
  - `3단계` — 서류 자동 도출 온보딩
  - `2중` — 회계사·매니저 검증 게이트

### 3.4 서비스 여정 (4스텝 플로우)
- 상단 라벨 `서비스 여정`(`14px/700/#2563EB`) + H2 `입력부터 정산완료까지, 한 흐름으로`(`32px/700`). 패딩 `72px 56px`.
- 가로 4스텝, 사이에 `→` 화살표 아이콘(`#CBD5E1`). 각 스텝: `56×56` 파랑tint(`#EFF6FF`) radius `15px` 아이콘칩 + 제목(`17px/700`) + 2줄 설명(`14px/#475569`):
  1. **사업계획서 입력** (문서 아이콘) — 업로드하거나 붙여넣으면 AI가 바로 분석합니다
  2. **필요서류 자동 도출** (톱니 아이콘) — 비목을 추정해 서류 체크리스트를 만듭니다
  3. **제출·이중검증** (방패 아이콘) — 회계사·매니저 이중 게이트로 검토
  4. **정산완료** (체크 아이콘) — 집행부터 완료까지 상태를 실시간 추적

### 3.5 핵심 기능 카드 (3×2 그리드)
- 라벨 `핵심 기능` + H2 `창업자와 매니저, 양방향으로`. `repeat(3,1fr)`, gap `20px`, 패딩 `12px 56px 76px`.
- 카드: 흰 배경, 테두리, 표준 그림자, padding `26px`. 상단에 번호(`22px/800/#CBD5E1`) + 역할 뱃지(창업자=파랑tint / 매니저=보라tint). 제목 `20px/700`, 본문 `15px/#475569`, 하단 `자세히 →` 링크(파랑).
  1. `01` 창업자 — **서류 자동 도출** — 사업계획서만 넣으면 필요한 서류와 표준양식을 자동으로 정리해 드립니다.
  2. `02` 창업자 — **비목 사전검증** — 부적합 비목·한도 초과·증빙 누락을 제출 전에 인라인으로 잡아줍니다.
  3. `03` 창업자 — **상태 트래커** — 초안부터 정산완료까지 파이프라인으로 한눈에 추적합니다.
